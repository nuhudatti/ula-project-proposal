What is an API?

An Application Programming Interface (API) is a set of rules that allows different software applications to communicate with each other. It defines how requests should be made and how responses will be structured.

Real-world example:

Think of an API like a waiter in a restaurant. You (client) tell the waiter (API) what you want, they communicate with the kitchen (server), and bring back your food (response).

Why APIs Matter

  • Efficiency: Reuse existing functionality instead of building from scratch
  • Integration: Connect different systems and services
  • Innovation: Build new services on existing platforms

Common API Types

WEB

Web APIs

Used for web services (like NITDA ULA API). Communicate over HTTP/HTTPS.

LIB

Library APIs

Interfaces to software libraries (like Python's requests library).

OS

OS APIs

Allow applications to interact with operating system functions.

API Communication Example 
Client Application → API Request → Server
Client Application ← API Response ← Server

REST API Basics

REST (Representational State Transfer) is the most common architecture style for web APIs. It uses standard HTTP methods to perform operations on resources.

Key Principles

  • 1 Client-Server: Separation of concerns between UI and data storage
  • 2 Stateless: Each request contains all necessary information
  • 3 Cacheable: Responses can be cached for better performance
  • 4 Uniform Interface: Consistent way to interact with the API

RESTful Resources

In REST, everything is a resource identified by a URI:

GET /api/students
GET /api/students/123
POST /api/students
PUT /api/students/123

NITDA Standard: All ULA APIs follow RESTful principles with JSON payloads for maximum compatibility and ease of use.

Your First API Call

Let's make a real API call to the NITDA ULA demo endpoint. We'll use Python with the requests library.

Steps to Follow

  1. 1 Install Python (if not already installed)
  2. 2 Install the requests library: pip install requests
  3. 3 Write and execute the code example
Python Example 
import requests

# NITDA ULA Demo API endpoint
url = "https://api.ula.nitda.gov.ng/v1/demo"

# Make GET request
response = requests.get(url)

# Check if request was successful
if response.status_code == 200:
    # Print the JSON response
    print(response.json())
else:
    print(f"Error: {response.status_code}")

HTTP Methods

HTTP methods define the type of operation you want to perform on a resource. These are the fundamental building blocks of REST APIs.

GET

GET

Retrieve data from the server. Should not modify any resources.

requests.get(url)

POST

POST

Create a new resource or submit data to be processed.

requests.post(url, json=data)

PUT

PUT

Update an existing resource or create if it doesn't exist.

requests.put(url, json=data)

DELETE

DELETE

Remove a specified resource.

requests.delete(url)

Example Usage

Python Examples 
# GET - Retrieve student with ID 123
requests.get("https://api.ula.nitda.gov.ng/v1/students/123")

# POST - Create new student
new_student = {"name": "Amina", "course": "Computer Science"}
requests.post("https://api.ula.nitda.gov.ng/v1/students", json=new_student)

# PUT - Update student record
updated_data = {"course": "Data Science"}
requests.put("https://api.ula.nitda.gov.ng/v1/students/123", json=updated_data)

# DELETE - Remove student
requests.delete("https://api.ula.nitda.gov.ng/v1/students/123")

HTTP Status Codes

Status codes are 3-digit numbers returned by servers to indicate the result of a request. They help you understand what happened with your API call.

2xx Success

  • 200 OK - Request succeeded
  • 201 Created - Resource created
  • 204 No Content - Success but no response

3xx Redirection

  • 301 Moved Permanently
  • 304 Not Modified - Use cached version

4xx Client Errors

  • 400 Bad Request - Invalid syntax
  • 401 Unauthorized - Authentication needed
  • 404 Not Found - Resource doesn't exist

5xx Server Errors

  • 500 Internal Server Error
  • 503 Service Unavailable

Handling Status Codes

Python Example 
response = requests.get("https://api.ula.nitda.gov.ng/v1/data")

if response.status_code == 200:
    # Success - process data
    data = response.json()
elif response.status_code == 401:
    # Authentication error
    print("Please authenticate first")
elif response.status_code == 404:
    # Not found
    print("Resource not found")
else:
    # Other errors
    print(f"Error: {response.status_code}")
