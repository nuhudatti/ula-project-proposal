1. Advanced Widget Configuration

Professional applications need polished interfaces. Tkinter provides extensive styling options:

Key Styling Options

font Control font family, size and weight
fg/bg Set foreground (text) and background colors
padx/pady Add padding around widgets
relief Create 3D border effects (flat, raised, sunken)

Styled Widget Example

Example of a styled button with hover effects

styled_widgets.py 
import tkinter as tk

window = tk.Tk()
window.title("Styled Widgets")

# Configure widget styling
button = tk.Button(
    window,
    text="Click Me",
    font=("Helvetica", 12, "bold"),
    fg="white",
    bg="#4CAF50",
    padx=15,
    pady=5,
    relief="raised",
    borderwidth=3
).pack()

window.mainloop()
Parameter Effect
font=("Helvetica", 12, "bold") Sets font family, size and weight
fg="white" Text color (foreground)
bg="#4CAF50" Background color
padx=15, pady=5 Horizontal and vertical padding
relief="raised" 3D border effect

2. Multiple Windows and Dialogs

Real applications often need multiple windows for different functions.

Creating Secondary Windows

Use Toplevel() for additional windows instead of creating new Tk() instances.

popup_window.py 
import tkinter as tk

def open_popup():
    popup = tk.Toplevel(window)
    popup.title("Popup Window")
    tk.Label(popup, text="This is a secondary window!").pack()

window = tk.Tk()
window.title("Main Window")

tk.Button(window, text="Open Popup", 
          command=open_popup).pack()

window.mainloop()

Key Points

  • Toplevel() creates child windows
  • Always create widgets with the new window as parent
  • Child windows close when main window closes
  • Use grab_set() for modal dialogs

Common Dialogs

Dialog Usage Example
messagebox Show alerts and confirmations tk.messagebox.showinfo("Title", "Message")
filedialog File selection dialogs tk.filedialog.askopenfilename()
colorchooser Color picker dialog tk.colorchooser.askcolor()

3. Data Validation

Ensure users enter correct data with input validation.

Validation Techniques

  • Key validation: Check each keystroke
  • Focus validation: Validate when leaving a field
  • Submit validation: Check all fields on form submission

Validation Example

Please enter numbers only!

validation.py 
import tkinter as tk

def validate_number(input):
    # Allow empty field or digits only
    if input.isdigit() or input == "":
        return True
    else:
        return False

window = tk.Tk()
window.title("Validation Example")

# Register validation function
validate_cmd = window.register(validate_number)

tk.Label(window, text="Enter a number:").pack()
entry = tk.Entry(
    window, 
    validate="key",  # Validate on each keystroke
    validatecommand=(validate_cmd, "%P")  # %P = new value
)
entry.pack()

window.mainloop()
Component Purpose
validate="key" Triggers validation on each keystroke
validatecommand Calls our validation function
%P Passes the new input value

4. Menu Systems

Professional applications need menu bars for navigation.

menu_example.py 
import tkinter as tk
from tkinter import messagebox

def file_new():
    messagebox.showinfo("New", "Creating new file...")

window = tk.Tk()
window.title("Menu Example")

# Create main menu bar
menubar = tk.Menu(window)

# File menu
file_menu = tk.Menu(menubar, tearoff=0)
file_menu.add_command(label="New", command=file_new)
file_menu.add_command(label="Open")
file_menu.add_separator()
file_menu.add_command(label="Exit", command=window.quit)
menubar.add_cascade(label="File", menu=file_menu)

# Edit menu
edit_menu = tk.Menu(menubar, tearoff=0)
edit_menu.add_command(label="Cut")
edit_menu.add_command(label="Copy")
edit_menu.add_command(label="Paste")
menubar.add_cascade(label="Edit", menu=edit_menu)

# Attach menu to window
window.config(menu=menubar)
window.mainloop()

Menu Components

  • Menu() - Creates a menu container
  • add_command() - Adds clickable menu item
  • add_separator() - Adds a dividing line
  • add_cascade() - Creates dropdown section
  • tearoff=0 - Disables menu detaching

Expected Output

File
Edit

5. Complete Intermediate Example

A note-taking application combining all intermediate concepts:

note_taker.py 
import tkinter as tk
from tkinter import messagebox

def save_note():
    note_text = text_area.get("1.0", "end-1c")
    if not note_text.strip():
        messagebox.showwarning("Empty Note", 
                             "Please write something before saving!")
    else:
        with open("notes.txt", "a") as f:
            f.write(note_text + "\n---\n")
        messagebox.showinfo("Success", "Note saved successfully!")

def clear_note():
    text_area.delete("1.0", "end")

def about_app():
    about_win = tk.Toplevel(window)
    about_win.title("About")
    tk.Label(about_win, 
            text="Note Taker v1.0\nBuilt with Tkinter").pack()

window = tk.Tk()
window.title("Note Taker")
window.geometry("500x400")

# Menu Bar
menubar = tk.Menu(window)

# File Menu
file_menu = tk.Menu(menubar, tearoff=0)
file_menu.add_command(label="Save", command=save_note)
file_menu.add_command(label="Clear", command=clear_note)
file_menu.add_separator()
file_menu.add_command(label="Exit", command=window.quit)
menubar.add_cascade(label="File", menu=file_menu)

# Help Menu
help_menu = tk.Menu(menubar, tearoff=0)
help_menu.add_command(label="About", command=about_app)
menubar.add_cascade(label="Help", menu=help_menu)

window.config(menu=menubar)

# Text Area
text_area = tk.Text(
    window,
    font=("Arial", 12),
    padx=10,
    pady=10,
    wrap="word"  # Wrap at word boundaries
)
text_area.pack(fill="both", expand=True)

window.mainloop()

Key Features

  • Styled text area with padding and word wrap
  • Menu bar with File and Help options
  • Validation preventing empty notes
  • Secondary window for About dialog
  • File operations saving notes to disk

How It Works

  1. User writes note in text area
  2. Clicking "Save" validates and saves to file
  3. "Clear" empties the text area
  4. "About" shows program information
