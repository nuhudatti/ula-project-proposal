Custom Widget Styling

Themed Tkinter (ttk)

The ttk module provides modern, themeable widgets that match native system styling. Key benefits:

  • Consistent look across platforms
  • Professional styling options
  • Better performance than standard widgets

Style Configuration

style = ttk.Style()
# Configure default style
style.configure("TButton", font=("Arial", 10), padding=6)

# Configure dynamic states
style.map("TButton",
    foreground=[('pressed', 'red'), ('active', 'blue')],
    background=[('pressed', '!disabled', 'black'), ('active', 'white')]
)

# Change global theme
style.theme_use("clam")

Example: Styled Notebook

styled_widgets.py 
import tkinter as tk
from tkinter import ttk

root = tk.Tk()
root.title("Professional Styling")

# Create and configure style
style = ttk.Style()
style.configure("TButton", font=("Arial", 10), padding=6)
style.configure("TNotebook.Tab", 
               font=("Arial", 10, "bold"), 
               padding=[10, 5])

# Create themed widgets
notebook = ttk.Notebook(root)
tab1 = ttk.Frame(notebook)
tab2 = ttk.Frame(notebook)

notebook.add(tab1, text="Dashboard")
notebook.add(tab2, text="Settings")
notebook.pack(expand=True, fill="both")

ttk.Button(tab1, text="Styled Button").pack(pady=10)
root.mainloop()

Threading with GUIs

Responsive UI with Threads

GUI applications become unresponsive during long-running tasks. Threading keeps your interface smooth while background tasks execute.

Critical Rule

Never update the GUI from non-main threads. Use after() or queues to communicate between threads.

Threading Example

threaded_gui.py 
import tkinter as tk
import threading
import time

def long_task():
    """Simulate a 3-second task"""
    time.sleep(3)
    # Safely update GUI from main thread
    root.after(0, lambda: status_label.config(text="Task Completed!"))

def start_thread():
    """Start the task in a background thread"""
    threading.Thread(target=long_task, daemon=True).start()
    status_label.config(text="Working...")

root = tk.Tk()
tk.Button(root, text="Run Task", command=start_thread).pack(pady=10)
status_label = tk.Label(root, text="Ready")
status_label.pack()
root.mainloop()

Threading Best Practices

  • Use daemon threads (daemon=True) for tasks that should exit with the application

  • Queue communication between threads for data transfer

  • Use locks when accessing shared resources

  • Progress feedback using progress bars or status labels

Thread Safety Pattern

import queue

# Create a queue for thread communication
gui_queue = queue.Queue()

def worker_thread():
    """Background worker thread"""
    result = do_long_calculation()
    gui_queue.put(result)

def check_queue():
    """Check for messages from threads"""
    try:
        msg = gui_queue.get_nowait()
        update_gui(msg)
    except queue.Empty:
        pass
    root.after(100, check_queue)  # Check every 100ms

# Start the queue checker
root.after(100, check_queue)

Database Patterns

  • CRUD Operations: Create, Read, Update, Delete records

  • Connection Pooling: Reuse database connections efficiently

  • Error Handling: Manage SQL exceptions gracefully

  • Thread Safety: Use separate connections per thread

Database Class Example

class Database:
    def __init__(self, db_file):
        self.db_file = db_file
        self.conn = None
        
    def connect(self):
        """Establish database connection"""
        self.conn = sqlite3.connect(self.db_file)
        self.conn.row_factory = sqlite3.Row
        
    def close(self):
        """Close the connection"""
        if self.conn:
            self.conn.close()
            
    def execute(self, query, params=()):
        """Execute a query with parameters"""
        cursor = self.conn.cursor()
        cursor.execute(query, params)
        self.conn.commit()
        return cursor

Packaging for Distribution

Creating Standalone Executables

Package your Tkinter application into a single executable file that runs without requiring Python installation.

Using PyInstaller

Terminal 
# Install PyInstaller
pip install pyinstaller

# Create single executable
pyinstaller --onefile --windowed --icon=app.ico app.py

# Add additional files
pyinstaller --add-data "data/*.db;data" app.py

Packaging Options

Flag Effect
--onefile Single executable file
--windowed No console window
--icon=app.ico Custom application icon
--add-data Include non-Python files
--name Set output name

Packaging Checklist

  • Test on clean virtual environment
  • Include all data files
  • Set proper file permissions
  • Create installer for end users

database.py 
import sqlite3
from contextlib import contextmanager

class InventoryDB:
    def __init__(self, db_file):
        self.db_file = db_file
        self._create_tables()
    
    @contextmanager
    def _get_connection(self):
        """Context manager for database connections"""
        conn = sqlite3.connect(self.db_file)
        conn.row_factory = sqlite3.Row
        try:
            yield conn
        finally:
            conn.close()
    
    def _create_tables(self):
        """Initialize database tables"""
        with self._get_connection() as conn:
            conn.execute("""CREATE TABLE IF NOT EXISTS products (
                        id INTEGER PRIMARY KEY,
                        name TEXT NOT NULL,
                        quantity INTEGER,
                        price REAL)""")
    
    def add_product(self, name, quantity, price):
        """Add a new product to inventory"""
        with self._get_connection() as conn:
            cursor = conn.cursor()
            cursor.execute("""INSERT INTO products (name, quantity, price)
                           VALUES (?, ?, ?)""", 
                           (name, quantity, price))
            conn.commit()
