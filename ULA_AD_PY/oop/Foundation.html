👨‍🏫 Object-Oriented Programming (OOP)

Why OOP? OOP helps us build real-world systems in code by modeling objects like people, vehicles, students — anything that has data (attributes) and behavior (methods).

🔹 1. Classes & Objects

A class is like a blueprint — it defines what an object should be. An object is the actual thing built from that blueprint.

# Class and Object
class Person:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def welcome(self):
        print(f"Hello {self.name}, age {self.age}, welcome to Python Advance class")

p1 = Person("Amina", 12)
p1.welcome()

We created a Person class with two properties: name and age. The __init__ method is a constructor — it runs automatically when we create a new object (like Amina). The welcome method prints a personalized message.

🔹 2. Methods

Methods are functions defined inside a class — they describe what the object can do.

🔹 3. Inheritance (Parent → Child)

Inheritance allows one class (child) to use the properties and methods of another (parent).

# Inheriting from Person
class Student(Person):
    def __init__(self, name, age, gradyear):
        super().__init__(name, age)
        self.gradyear = gradyear

    def attendance(self):
        checkin = input("Enter Check-In time: ")
        checkout = input("Enter Check-Out time: ")
        print(f"Hello {self.name}, class of {self.gradyear}, checked in at {checkin} and out at {checkout}.")

stu = Student("John", 21, 2027)
stu.attendance()

The Student class inherits from the Person class, gaining access to its properties and methods. super() allows us to call the constructor of the parent class. attendance() is a new method just for Student.

🔹 4. Polymorphism

Polymorphism means different classes can define their own version of a method with the same name. The function runs based on the object’s class.

class Car:
    def move(self):
        print("DRIVE!!")

class Boat:
    def move(self):
        print("SAIL!!")

class Plane:
    def move(self):
        print("FLY!!")

# Demonstrating polymorphism
for x in (Car(), Boat(), Plane()):
    x.move()

Each class has a method called move(), but each one behaves differently. This is called polymorphism — one method name, different implementations based on object type.

🔹 5. Overriding Methods

You can override a parent class’s method in the child class.

class Vehicle:
    def move(self):
        print("MOVE!!")

class Boat(Vehicle):
    def move(self):
        print("SAIL!!")

Boat inherits from Vehicle, but it overrides the move() method. This means it replaces the parent behavior with its own version — this is very useful when child objects need unique actions.

🌟 Summary

  • ✅ Classes group related data and actions
  • ✅ Methods are object-specific functions
  • ✅ Inheritance lets you reuse and extend code
  • ✅ Polymorphism makes your code dynamic and adaptable
  • ✅ Override lets you customize behavior in child classes
