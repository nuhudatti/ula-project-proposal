üíª Abstract Class in Action

from abc import ABC, abstractmethod

class Animal(ABC):
    # Abstract method
    @abstractmethod
    def sound(self):
        pass

class Dog(Animal):
    def sound(self):
        print("Bark!")

class Cat(Animal):
    def sound(self):
        print("Meow!")

# Instantiate objects
d = Dog()
d.sound()

c = Cat()
c.sound()

üî∏ Animal is an abstract class ‚Äî it only gives the idea of a sound, not the exact sound.

üî∏ @abstractmethod forces child classes like Dog and Cat to define their own sound() method.

üß† This makes your code blueprint-driven and extensible ‚Äî just like real-world contracts.

üè¶ Real-Life Analogy: Payment System

from abc import ABC, abstractmethod

class Payment(ABC):
    @abstractmethod
    def pay(self, amount):
        pass

class CashPayment(Payment):
    def pay(self, amount):
        print(f"Paying ‚Ç¶{amount} in cash.")

class CardPayment(Payment):
    def pay(self, amount):
        print(f"Paying ‚Ç¶{amount} using card.")

cp = CashPayment()
cp.pay(500)

card = CardPayment()
card.pay(1000)

üîπ Payment is an abstract base for different payment methods.

üîπ Each class implements pay() differently, but follows the same blueprint.

üíº Abstraction like this powers apps like Flutterwave, Paystack, and Stripe.
