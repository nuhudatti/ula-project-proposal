Python Indentation

Python uses spaces/tabs to organize code (unlike other languages that use braces {}). This makes Python code very readable but requires careful formatting.

Correct Indentation

if 5 > 2:
    print("Five is greater than two!")

Wrong Indentation

if 5 > 2:
print("Five is greater than two!")  # Error!

Understanding Indentation

Think of indentation like paragraphs in a book:

  • No indentation = New thought (new block of code)
  • Indented = Part of the previous thought (code block)

Real World Example

Imagine giving instructions:

1. Go to the kitchen

- Open the fridge

- Take the milk

2. Come back to the living room

The indented steps belong to "Go to the kitchen" just like indented code belongs to an if statement.

Common Mistake

Mixing tabs and spaces causes errors. Configure your editor to use 4 spaces per tab.

Variables are like labeled boxes where you store information. You give each box a name and put data inside it.

age = 25 Creates a variable storing a whole number Like writing your age on a form
is_student = True Stores a True/False value Like checking a yes/no box

Variable Naming Rules

  • Must start with a letter or underscore
  • Can contain letters, numbers, and underscores

5. Operators & Expressions

Basic Math Operators

Python can perform calculations just like a calculator. Here are the basic math operators:

Operator Example Result Real-life Example
+ 5 + 3 8 Adding apples
- 5 - 3 2 Subtracting expenses
* 5 * 3 15 Multiplying items
/ 5 / 2 2.5 Dividing pizza slices
% 5 % 2 1 Remainder after division
** 2 ** 3 8 2 to the power of 3

Practical Examples

operators.py 
# Calculate total price
price = 15
quantity = 3
total = price * quantity
print("Total:", total)  # Output: Total: 45

# Calculate area of a circle
radius = 5
area = 3.14 * (radius ** 2)
print("Area:", area)  # Output: Area: 78.5

# Check if number is even
number = 10
remainder = number % 2
print("Remainder:", remainder)  # Output: Remainder: 0

Operator Precedence

Python follows math rules for operation order:

  1. Parentheses ()
  2. Exponents **
  3. Multiplication/Division * / // %
  4. Addition/Subtraction + -
result = 5 + 3 * 2   # = 11 (not 16)
result = (5 + 3) * 2 # = 16

Comparison Operators

These operators compare values and return True or False:

Operator Meaning Example Result
== Equal to 5 == 5 True
!= Not equal 5 != 3 True
> Greater than 5 > 3 True
< Less than 5 < 3 False
comparisons.py 
age = 18
is_adult = age >= 18  # Check if age is 18 or more
print("Is adult?", is_adult)  # Output: Is adult? True

temperature = 22.5
is_hot = temperature > 30
print("Is hot?", is_hot)  # Output: Is hot? False

Practical Tip

Use comparison operators to make decisions in your code. For example: if temperature > 30: print("It's hot!")

6. Control Flow (Conditionals & Loops)

If-Else Statements

If-else statements let your program make decisions based on conditions.

conditions.py 
age = 18

if age >= 18:
    print("You are an adult.")
else:
    print("You are a minor.")

How If-Else Works

Code Explanation Real-life Example
if age >= 18: Checks if age is 18 or more Like checking ID for voting
print("Adult") Runs if condition is True "You can vote"
else: Runs if condition is False "You can't vote yet"

Elif for Multiple Conditions

score = 85

if score >= 90:
    grade = "A"
elif score >= 80:
    grade = "B"
elif score >= 70:
    grade = "C"
else:
    grade = "F"
    
print("Your grade:", grade)  # Output: Your grade: B

Loops

Loops repeat code until a condition is met. Python has two main loop types:

For Loop

Runs code for each item in a sequence:

# Print numbers 0 to 4
for i in range(5):
    print(i)

# Loop through a list
fruits = ["apple", "banana", "cherry"]
for fruit in fruits:
    print("I like", fruit)

While Loop

Runs code while condition is True:

# Countdown from 5
count = 5
while count > 0:
    print(count)
    count -= 1  # Same as count = count - 1
print("Blastoff!")

Understanding Loops

For Loop

  • Used when you know how many times to repeat
  • Works with sequences (lists, strings, ranges)
  • range() generates numbers
# range(5) produces 0, 1, 2, 3, 4
for x in range(5):
    print(x * 2)  # Prints 0, 2, 4, 6, 8

While Loop

  • Used when you don't know how many times to repeat
  • Continues until condition becomes False
  • Must update condition variable in loop
# User input example
password = ""
while password != "secret":
    password = input("Enter password: ")
print("Access granted!")

Important Warning

Avoid infinite loops! Always ensure your while loop condition will eventually become False.

# Infinite loop - don't do this!
while True:
    print("This will run forever!")

Loop Control

Use break to exit a loop early, or continue to skip to the next iteration.

for num in range(10):
    if num == 3:
        continue  # Skip 3
    if num == 7:
        break     # Stop at 7
    print(num)     # Prints 0,1,2,4,5,6

7. Functions

Creating Functions

Functions are reusable blocks of code that perform specific tasks. They help organize your code and avoid repetition.

functions.py 
# Define a simple function
def greet():
    print("Hello there!")
    
# Call the function
greet()  # Output: Hello there!

# Function with parameters
def greet_person(name):
    print("Hello", name)
    
greet_person("Alice")  # Output: Hello Alice

Function Components

Part Purpose Example
def Starts function definition def my_function():
Function name How you call the function calculate_total()
Parameters Variables passed to the function (price, quantity)
Return value Result sent back from function return total

Returning Values

# Function that returns a value
def calculate_area(length, width):
    area = length * width
    return area

# Store the returned value
room_area = calculate_area(10, 12)
print("Room area:", room_area, "sq ft")

Function Parameters

Parameters make functions more flexible by allowing you to pass different values each time you call the function.

Positional Arguments

Values are matched by position:

def describe_pet(animal, name):
    print(f"I have a {animal} named {name}")

describe_pet("dog", "Rex")  # Correct order
describe_pet("Rex", "dog")  # Wrong order!

Keyword Arguments

Explicitly name each argument:

describe_pet(animal="cat", name="Whiskers")
describe_pet(name="Buddy", animal="parrot")  # Order doesn't matter
default_args.py 
# Default parameter values
def describe_pet(name, animal="dog"):
    print(f"I have a {animal} named {name}")

describe_pet("Rex")          # Uses default animal
describe_pet("Fluffy", "cat")  # Overrides default

Function Best Practices

  • Keep functions small and focused on one task
  • Use descriptive names (verbs for actions)
  • Include docstrings to explain what the function does
  • Limit parameters to 3-4 when possible

8. Lists & Dictionaries

Working with Lists

Lists are ordered collections of items that can hold different data types. They are mutable (can be changed).

lists.py 
# Creating lists
fruits = ["apple", "banana", "cherry"]
numbers = [1, 2, 3, 4, 5]
mixed = [1, "hello", 3.14, True]

# Accessing elements
print(fruits[0])   # Output: apple (first item)
print(numbers[-1])  # Output: 5 (last item)

# Modifying lists
fruits[1] = "blueberry"  # Change second item
fruits.append("orange")  # Add to end
fruits.remove("apple")   # Remove item

Common List Operations

Operation Example Result
Length len(fruits) 3
Slicing numbers[1:3] [2, 3]
Concatenation [1,2] + [3,4] [1,2,3,4]
Membership "banana" in fruits True

List Methods

nums = [5, 2, 8, 1]

nums.sort()          # [1, 2, 5, 8]
nums.reverse()       # [8, 5, 2, 1]
nums.pop()           # Removes and returns last item (1)
nums.insert(1, 10)   # Insert 10 at position 1
nums.clear()         # Empty the list

Understanding Dictionaries

Dictionaries store key-value pairs and allow fast lookups by key (like a real dictionary).

dictionaries.py 
# Creating dictionaries
student = {
    "name": "Alice",
    "age": 20,
    "courses": ["Math", "Physics"]
}

# Accessing values
print(student["name"])  # Output: Alice
print(student.get("age"))  # Safer access

# Modifying dictionaries
student["age"] = 21          # Update value
student["grade"] = "A"       # Add new key
del student["courses"]       # Remove key

Dictionary Methods

# Get all keys
keys = student.keys()

# Get all values
values = student.values()

# Get key-value pairs
items = student.items()

# Remove and return item
age = student.pop("age")

Dictionary Iteration

# Loop through keys
for key in student:
    print(key, student[key])

# Loop through items
for key, value in student.items():
    print(f"{key}: {value}")

Data Structure Tips

  • Use lists when order matters and you need duplicates
  • Use dictionaries for fast lookups by key
  • Lists use square brackets [], dictionaries use curly braces {}
  • Both can be nested (lists in dictionaries, etc.)

9. File Handling

Reading and Writing Files

Python makes it easy to work with files on your computer. The basic modes are:

Mode Description Example
'r' Read (default) open('file.txt', 'r')
'w' Write (overwrites) open('file.txt', 'w')
'a' Append open('file.txt', 'a')
'r+' Read and write open('file.txt', 'r+')

Reading Files

# Read entire file
with open('data.txt') as file:
    content = file.read()
    print(content)

# Read line by line
with open('data.txt') as file:
    for line in file:
        print(line.strip())

Writing Files

# Write to file (overwrites)
with open('output.txt', 'w') as file:
    file.write("Hello World!\n")
    file.write("Second line\n")

# Append to file
with open('output.txt', 'a') as file:
    file.write("Appended line\n")

Working with CSV Files

CSV (Comma Separated Values) is a common format for spreadsheet data.

csv_example.py 
import csv

# Reading CSV
with open('data.csv') as csvfile:
    reader = csv.reader(csvfile)
    for row in reader:
        print(row)  # Each row is a list

# Writing CSV
with open('output.csv', 'w', newline='') as csvfile:
    writer = csv.writer(csvfile)
    writer.writerow(["Name", "Age", "City"])
    writer.writerow(["Alice", 25, "New York"])
    writer.writerow(["Bob", 30, "Chicago"])

File Handling Best Practices

  • Always use with statement for automatic file closing
  • Handle potential exceptions (FileNotFoundError, etc.)
  • For large files, read line by line instead of all at once
  • Be careful with write mode ('w') as it overwrites

10. Error Handling

Try-Except Blocks

Handle errors gracefully to prevent your program from crashing unexpectedly.

error_handling.py 
# Basic error handling
try:
    age = int(input("Enter your age: "))
    print(f"You are {age} years old")
except ValueError:
    print("Please enter a valid number!")

# Handling multiple exceptions
try:
    num = 10 / 0
    open("nonexistent.txt")
except ZeroDivisionError:
    print("Can't divide by zero!")
except FileNotFoundError:
    print("File not found!")

Else and Finally

try:
    result = 10 / 2
except ZeroDivisionError:
    print("Division failed")
else:
    print("Result is", result)  # Runs if no error
finally:
    print("This always runs")  # Cleanup code

Raising Exceptions

def validate_age(age):
    if age < 0:
        raise ValueError("Age can't be negative!")
    elif age < 18:
        raise ValueError("Must be 18 or older")
    else:
        print("Age is valid")

try:
    validate_age(-5)
except ValueError as e:
    print(e)  # Output: Age can't be negative!

Error Handling Tips

  • Be specific about which exceptions you catch
  • Use finally for cleanup code (closing files, etc.)
  • Create custom exceptions for your application
  • Log errors for debugging while showing user-friendly messages

Bonus. Recursion

Recursion is when a function calls itself to solve smaller versions of a problem.

Example: Factorial

def factorial(n):
    if n == 1:
        return 1
    return n * factorial(n - 1)

print(factorial(5))  # Output: 120

Use recursion carefully to avoid infinite loops.

Recursion

Topics Covered

  • Understanding base and recursive cases
  • Recursion vs iteration (loops)
  • Common recursive problems (e.g., factorial, Fibonacci)
  • Python recursion limit

Example


def factorial(n):
    if n == 0:
        return 1
    else:
        return n * factorial(n - 1)
